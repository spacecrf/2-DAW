Memoria técnica — CSS aplicado por apartados

En esta memoria se describe, de forma concisa, qué reglas CSS se aplican a cada bloque de la página y qué conceptos del temario se han utilizado.

Global y tipografía

  • box-sizing: border-box se aplica globalmente a todos los elementos para facilitar el modelo de cajas.
  • Unidades empleadas: rem (para fuentes y espacios), %, vw para medidas relativas y px cuando es necesario; ejemplo: font-size: 1.125rem;.
  • Pseudo-clases usadas: :hover en enlaces y botones; :focus en campos del formulario; pseudo-elementos ::before para decorar encabezados.

Header y menú

  • Menú horizontal tradicional implementado con una lista inline que se adapta mediante flexbox en pantallas grandes y con un botón que muestra/oculta (small) usando media queries.
  • Posicionamiento: position: sticky en la cabecera para mantener el menú visible al hacer scroll.

Hero

  • Diseño en dos columnas con grid (hero-grid). La imagen es un placeholder con altura en vh para responsividad.
  • Overflow controlado: el contenedor usa overflow: hidden para evitar solapamientos con elementos posicionados.

Instrumentos (cards)

  • Uso de display: grid en .grid-list con grid-template-columns: repeat(auto-fit, minmax(220px, 1fr)) para adaptabilidad.
  • Boxes y sombras, padding y margin aplicados con rem para mantener escalado coherente.

Galería — flotantes y clearfix

  • Se muestra un ejemplo deliberado con float: .float-item { float: left; width: 30%; margin-right: 2%; } para ilustrar problemas típicos (colapso del contenedor).
  • Solución con .clearfix::after (pseudo-elemento) y/o overflow: auto en el contenedor. En este proyecto se incluye la clase .clearfix.

Formularios

  • Estados con pseudo-clases: input:focus y button:hover para accesibilidad visual.
  • Se verifican required desde HTML y estilos que mejoran usabilidad.

Responsividad

  • Media queries en puntos clave: 900px y 600px para reorganizar nav, grid y hero.
  • Uso de unidades relativas como % y rem para que la interfaz escale.

Problemas previstos y soluciones

  • Problema espacio entre inline-block: resuelto usando flexbox y eliminando espacios en el HTML donde se emplea inline-block, o usando comentarios HTML si se usa inline-block.
  • Problemas con flotantes: uso de clearfix y alternativas modernas (grid/flex) recomendadas.
